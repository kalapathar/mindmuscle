State Specification

Abstract class used as a template to implement the states of the game.

Omar Shehata, 5/15




CLASS public State

No superclass.

State variables:

public string name
The name of this state

public string msg
A message that can be set to transition to another state

public int sent
To ensure the message is sent only once

Methods:

virtual void onEnter
Runs when the state is activated

No arguments.

State change:
To be implemented by state

Return value:
No return value.

virtual void onExit
Runs when the state is deactivated

No arguments.

State change:
To be implemented by state

Return value:
No return value.

virtual void render
Provides an interface for rendering on screen

No arguments.

State change:
To be implemented by state

Return value:
No return value.

virtual void update
Runs 60 times per second

No arguments.

State change:
To be implemented by state

Return value:
No return value.

virtual void keyboard
Provides an interface for key presses

Arguments:

unsigned char c

int x

int y

State change:
To be implemented by state

Return value:
None.

virtual void mouse
Provides an interface for mouse presses

Arguments:

int button

int state

int x

int y

State change:
To be implemented by state

Return value:
No return value.

virtual void mouse_motion
Provides an interface for the mouse movement

Arguments:

int x

int y

State change:
To be implemented by state

Return value:
No return value.
