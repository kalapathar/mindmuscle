GameObject Specification

This is the base class for all objects that are drawn on screen.

Omar Shehata. May 3 2015. Richard Brown. XML version of specification written by Aubrey Barnard, Wednesday, April 10, 2002




CLASS public GameObject
Represents an on-screen object.

No superclass.

State variables:

public float x
x coordinate on screen

public float y
y coordinate on screen

public float angle
The angle at which to draw the image

public float width
The width of the image

public float height
The height of the image

public * b2Body body
Pointer to physics body

private int texture
The ID of the texture associated with this object

private int depth
An ID representing the order that objects get drawn on screen

public static int numObjects
Static variable keeping track of how many objects were created

Constructors:

public GameObject
Regular constructor.

Arguments:

String filename
The filename to the image to be associated with this object

Boolean attachPhysics
Whether to attach a physics body

State change:
Initializes a GameObject by loading the texture given by filename, sets x and y to 0, creates a physics body if attachPhysics is true, and assigns it to body, otherwise assign it 0. Assigns width and height a default of 100. Sets depth to numObjects and increments numObjects by 1.

Destructor:
~GameObject
Destructor.

State change:
Deallocate the texture created and the physics body if exists.

Methods:

void setDepth
Changes the depth of the object

Arguments:

int d
An integer representing depth

State change:
Sets depth to d

Return value:
No return value.

void draw
Draws the object to screen

No arguments.

State change:
Handles all the OpenGL code needed to draw this object at the correct location, angle and scale.

Return value:
No return value.

void update
Binds the object to the physics body

No arguments.

State change:
Sets the x,y and angle to follow the attached physics body.

Return value:
No return value.
